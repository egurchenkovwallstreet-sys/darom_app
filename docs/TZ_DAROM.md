# Техническое задание: «Даром» v3.0

> Якорный файл проекта. В новом чате: `@docs/TZ_DAROM.md` + `@docs/PROGRESS.md`

## 1. Концепция

**«Даром»** — мобильное приложение для **бесплатной** передачи вещей. Единый аккаунт: пользователь одновременно отдаёт и забирает вещи. Узкая специализация, без коммерции. Геолокация — ключевая ценность.

**Приоритет:** Flutter (Android + iOS) → потом Flutter Web.

> ⚠️ **КРИТИЧНО — БЕЗОПАСНОСТЬ (раздел 13):** публичный запуск для **всех** пользователей **запрещён**, пока не выполнен **Этап I — защита** и пройден **обязательный чеклист** (см. `docs/PROGRESS.md` → «Безопасность»). Мягкий тест среди знакомых — допустим с пониманием рисков.

## 2. Роли и монетизация

### Бесплатно всегда
- Регистрация: **номер + имя + PIN** (без SMS при регистрации)
- Карта и лента объявлений
- До **30 активных объявлений** (единый лимит для всех)
- Профиль и статистика

### Подтверждение номера (один раз навсегда)
- **Не** при регистрации — только при **первом объявлении** или **первом сообщении в чате**
- Канал: **SMS Aero Mobile ID** (~3,39–5,79 ₽): push «Подтвердить» или SMS с 4 цифрами
- После успеха — все функции без повторной проверки
- Инструкция: `deploy/MOBILE_ID.md`

### Основатель (первые 1000 пользователей)
- Значок навсегда, приоритет в ленте
- **Лимит объявлений и заборы — как у всех** (30 объявлений бесплатно; бонус заборов только если зарегистрировался по реферальному коду блогера)

### Монетизация получателей

**Бесплатные заборы в месяц** (сброс каждый календарный месяц). Порог считается по числу **активных объявлений на платформе** (статусы `active` + `reserved`):

| Режим платформы | Условие | Обычный пользователь | Реферал блогера* |
|-----------------|---------|----------------------|------------------|
| Стартовый | **&lt; 20 000** активных объявлений | **5**/мес | **7**/мес |
| Средний | **≥ 20 000** и **&lt; 50 000** | **3**/мес | **5**/мес |
| Зрелый | **≥ 50 000** активных объявлений | **2**/мес | **2**/мес |

\* **Реферал блогера** — пользователь, зарегистрировавшийся с кодом партнёра/блогера (поле `referred_by_partner_id`). Бонус **+2** забора к лимиту обычного пользователя на стартовом и среднем этапах; после 50 000 объявлений на платформе — **без бонуса** (2 для всех).

**Платные пакеты** (после исчерпания бесплатных, **каждый месяц с начала**):

| Пакет | Цена | Заборов |
|-------|------|---------|
| 1-й | **149₽** | +10 |
| 2-й | **299₽** | +10 |
| 3-й | **499₽** | +10 |
| После 3-го | **блок** до нового месяца | — |

- Цель: защита от массового бесплатного забора для перепродажи на других площадках
- Пожизненный счётчик «забрано» **не обнуляется**
- «Активировать повторно» — лимит получателя **не** тратится
- Сделка только после «Отдал» дарителем

### Монетизация дарителей
- Базово: **30** активных объявлений **для всех** (основатель и обычный пользователь — одинаково)
- После **30** активных объявлений — платная услуга **«Супер даритель»**: **99₽** за пакет **+10** активных объявлений на **30 дней** (значок, приоритет в ленте)
- При исчерпании лимита пакет можно покупать **снова без ограничений**: каждые **99₽** → ещё **+10** мест (30→40→50→…)
- Пакеты **суммируются**: лимит = 30 + (число купленных пакетов × 10)

### 7.2 Сделка через чат (бронирование) — актуальная логика (27.06.2026)

> **Статус:** ✅ реализовано (backend + Flutter Web, 27.06.2026). Миграция: `migrate_chat_system_messages.sql`.

**Цель:** договориться в чате → забронировать → честно завершить сделку; защита от обхода лимита заборов (удаление объявления без «Отдал»).

| Шаг | Кто | Что происходит |
|-----|-----|----------------|
| 1 | Получатель | Пишет **первое сообщение** автору в чате по объявлению |
| 2 | Даритель (автор) | **Отвечает** хотя бы одним сообщением |
| 3 | Получатель | Становится активной кнопка **«Забронировать на 24 ч»** (до шагов 1–2 — **неактивна**) |
| 4 | Получатель | Нажимает «Забронировать» → объявление `reserved`, бронь 24 ч |
| 5 | Даритель | В чате появляется **системное сообщение** (точный текст): *«Вещь забронирована. После передачи обязательно отметьте, что сделка прошла успешно — так вам засчитается отданная вещь в рейтинг»* |
| 6 | Даритель | В **этом же чате** — две кнопки: **«Отдал вещь»** и **«Активировать повторно»** (дублируют логику карточки объявления) |
| 7a | Даритель → «Отдал вещь» | Сделка засчитана: дарителю +1 «отдано» / рейтинг; получателю +1 «забрано» и **−1 забор** (бесплатный или из пакета) |
| 7b | Даритель → «Активировать повторно» | Объявление снова **active**; забор получателя **не** списывается (как в §2) |

**Правила:**
- Сделка считается состоявшейся **только** после «Отдал вещь» (не от брони и не от удаления объявления).
- «Активировать повторно» — если вещь **не** передана (не приехали / отказ); защита от сценария «забронировали → удалили объявление → забор не списался».
- Push при брони дарителю ✅ (как сейчас).
