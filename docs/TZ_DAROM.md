# Техническое задание: «Даром» v3.0

> Якорный файл проекта. В новом чате: `@docs/TZ_DAROM.md` + `@docs/PROGRESS.md`

## 1. Концепция

**«Даром»** — мобильное приложение для **бесплатной** передачи вещей. Единый аккаунт: пользователь одновременно отдаёт и забирает вещи. Узкая специализация, без коммерции. Геолокация — ключевая ценность.

**Приоритет:** Flutter (Android + iOS) → потом Flutter Web.

## 2. Роли и монетизация

### Бесплатно всегда
- Регистрация: **номер + имя + PIN** (без SMS при регистрации)
- Карта и лента объявлений
- До **10 активных объявлений**
- Профиль и статистика

### Подтверждение номера (один раз навсегда)
- **Не** при регистрации — только при **первом объявлении** или **первом сообщении в чате**
- Канал: **SMS Aero Mobile ID** (~3,39–5,79 ₽): push «Подтвердить» или SMS с 4 цифрами
- После успеха — все функции без повторной проверки
- Инструкция: `deploy/MOBILE_ID.md`

### Основатель (первые 1000 пользователей)
- Значок навсегда, приоритет в ленте, **20 объявлений**
- **Монетизация как у всех** (без льгот на заборы)

### Монетизация получателей

**Бесплатные заборы в месяц** (сброс каждый календарный месяц):

| Режим платформы | Условие | Бесплатно/мес |
|-----------------|---------|---------------|
| Стартовый | **&lt; 20 000** активных объявлений на платформе | **7** |
| Полноценный старт | **≥ 20 000** активных объявлений | **3** |

**Платные пакеты** (после исчерпания бесплатных, **каждый месяц с начала**):

| Пакет | Цена | Заборов |
|-------|------|---------|
| 1-й | **149₽** | +10 |
| 2-й | **299₽** | +10 |
| 3-й | **499₽** | +10 |
| После 3-го | **блок** до нового месяца | — |

- Цель: защита от массового бесплатного забора для перепродажи на других площадках
- Пожизненный счётчик «забрано» **не обнуляется**
- «Активировать повторно» — лимит получателя **не** тратится
- Сделка только после «Отдал» дарителем

### Монетизация дарителей
- Базово: **10** активных объявлений (у **основателя** — **20**)
- **Супер даритель:** **99₽** за пакет **+10** активных объявлений на **30 дней** (значок, приоритет в ленте)
- При исчерпании лимита пакет можно покупать **снова без ограничений**: каждые **99₽** → ещё **+10** мест (20→30→40→… у обычного; 30→40→50→… у основателя)
- Пакеты **суммируются**: лимит = базовый + (число купленных пакетов × 10)

### Оплата
Робокасса (карты, СБП, кошельки). Нужно ИП/ООО.

**Статус (21.06.2026):** интеграция в коде ✅ (`deploy/ROBOKASSA.md`). Магазин **Darom-app** в кабинете Робокассы **на одобрении** — боевой тест оплаты **после активации**. Пока `PAYMENT_MOCK=true` или тестовая активация без списания.

### Партнёры / блогеры (реферальная программа)

| Правило | Описание |
|---------|----------|
| Регистрация партнёра | Код **0001–1000** по очереди; кнопка «Я партнёр / блогер» |
| Заявка на партнёрство | Почта **Darom.partner.ru@yandex.ru** |
| Код для аудитории | Номер партнёра (0001, 0002…) при регистрации пользователя |
| Привязка реферала | **365 дней** с регистрации |
| Комиссия | **30%** со **всех оплат** реферала в период 365 дней |
| После 365 дней | Реферал не в статистике, новые оплаты не идут |
| Выплаты | Ежемесячно; «за месяц» обнуляется после оплаты админом |
| Всего заработано | Накопительная сумма, не обнуляется |
| Статус | ✅ backend + Flutter + админ-UI «Блогеры» |

### Админ-панель

| Правило | Описание |
|---------|----------|
| URL | `/admin` (Flutter Web) — запасной путь |
| **Вход из профиля** | Пункт «Админ-панель» в ЛК — **только** если телефон есть в `admin_users` (проверка на backend: `can_access_admin_panel`) |
| **После кнопки** | Телефон уже известен → «Получить коды» → **реальное SMS** + код с почты → кабинет |
| Вход | **2FA:** **реальное SMS** на телефон super admin + код на email (SMTP ⏳ mock) |
| Super admin | Полный доступ: жалобы, блоки, статистика, блогеры, выплаты |
| Moderator (позже) | Только жалобы и блокировки — **без** монетизации и статистики |
| Жалобы объявлений | С контекстом объявления |
| Жалобы чатов | Полная переписка; кнопка «Пожаловаться на чат» в каждом чате |
| Блокировка | Временная **1–7 дней** (ручной выбор) или **навсегда** |
| Статистика | Пользователи, активные объявления, оплаты — периоды: день/неделя/месяц/всего |
| Блогеры | Следующий код (копировать), по каждому: телефон, рефералы, суммы, кнопка «Оплатить» |
| SMTP кодов | ⏳ mock (код в логах backend); боевой SMTP позже |
| Статус | ✅ backend + Flutter UI + вход из профиля |

## 7. Системные правила

- **Модерация:** Vision ⏳ + стоп-слова ✅; 3 жалобы → скрытие ✅
- **Рейтинг:** 1–5 после сделки ✅; <4.0 → теневой бан ✅ (backend)
- **Лимиты:** объявления 10/20 + Супер даритель (+10 за 99₽) ✅; заборы 7→3/мес + пакеты **149→299→499₽** ✅
- **Клавиатура на Web:** поля ввода не перекрываются (`KeyboardInsetPadding` / `AuthFormScroll`) ✅
- **Безопасность:** 1 телефон = 1 аккаунт ✅; PIN 4 цифры ✅; подтверждение реального номера один раз (Mobile ID) ✅; предупреждение о номере в чате ✅; блокировка пользователей ✅; админ — 2FA (реальное SMS + почта mock) ✅
- **Геолокация Web:** запрос на HTTPS (darom-app.online) ✅; подсказки и «Повторить» при отказе ✅
- **Чаты:** в списке только диалоги с ≥1 сообщением (пустые не показываются) ✅
- **Забронировано:** 24ч, серый статус ✅; push дарителю ⏳

### 7.1 Авторизация (актуальная логика, 21.06.2026)

| Этап | Что происходит | SMS / стоимость |
|------|----------------|-----------------|
| **Регистрация** | Номер → имя → PIN (4 цифры) | **Нет** |
| **Вход** | Номер + PIN | **Нет** |
| **Первое объявление или 1-е сообщение в чате** | Диалог «Подтвердить номер»; Mobile ID | **~3–6 ₽** (SMS Aero) |
| **Партнёр** | Код + телефон → Mobile ID → имя → PIN | **~3–6 ₽** (Mobile ID) |
| **Админ-панель** | SMS + код с почты | **Реальное SMS** + почта mock |
| **Сброс PIN** | SMS-код | Реальное SMS |

Поля в БД: `phone_verified_at` (аккаунт создан), `real_phone_verified_at` (номер подтверждён навсегда).

## 8. Уровни дарителей

Новичок → Активный (5+, рейтинг 4.0+) → Щедрый (20+) → Благотворитель (50+) → **Самое доброе сердце** (100+, 4.8+)

**В профиле (UI):** блок «🏆 Достижения» — **5 бейджей** по этим уровням (`lib/data/profile_achievements.dart`). Не полученный уровень — **маленький и серый**; полученный — **большой и цветной**. Счётчик: получено / 5. Логика совпадает с backend (`donor_level.js`).
