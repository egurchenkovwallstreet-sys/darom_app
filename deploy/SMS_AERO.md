# SMS Aero — подключение боевых SMS

Регистрация обычных пользователей остаётся **бесплатной** (тестовый код на экране).  
**Реальные SMS** уходят при:

- первом объявлении или первом сообщении в чате;
- регистрации партнёра/блогера;
- **не** при регистрации обычного пользователя.
- **да** при входе в админ-панель (реальное SMS на admin-телефон).
- **да** при регистрации партнёра (Mobile ID ~3–6 ₽).

## 1. Кабинет SMS Aero

1. [smsaero.ru](https://smsaero.ru) → **Настройки** → **API и SMPP**
2. Скопируйте **email** (логин) и **API-ключ**
3. Имя отправителя по умолчанию: **SMS Aero** (если другое — укажите в `SMS_AERO_SIGN`)

⚠️ API-ключ **не отправляйте** в чаты и **не коммитьте** в Git.

## 2. Сервер (VNC)

```bash
cd /opt/darom_app/backend
nano .env
```

Добавьте или измените строки:

```env
SMS_PROVIDER=smsaero
SMS_AERO_EMAIL=ваш_email@yandex.ru
SMS_AERO_API_KEY=ваш_api_ключ_из_кабинета
SMS_AERO_SIGN=SMS Aero
SMS_MOCK=false
```

Сохранить: `Ctrl+O`, Enter, `Ctrl+X`.

```bash
cd /opt/darom_app
git pull
pm2 restart darom-api --update-env
```

## 3. Проверка

1. Зарегистрируйте **нового** пользователя — код по-прежнему **на экране** (тест).
2. Попробуйте **опубликовать объявление** — должен прийти **реальный SMS** на телефон.
3. В кабинете SMS Aero → **История** — должна появиться отправка (~2–5 ₽).

## 4. Если SMS не приходит

| Проблема | Решение |
|----------|---------|
| Ошибка «sign» / отправитель | В кабинете SMS Aero проверьте одобренное имя отправителя |
| Баланс 0 | Пополните баланс |
| В логах `SMS mock (real requested…)` | Не задан ключ или `SMS_MOCK=true` |
| Ключ утёк | В кабинете **сгенерируйте новый API-ключ** |

Логи backend:

```bash
pm2 logs darom-api --lines 50
```

## Локальная разработка (ПК)

В `backend/.env` оставьте `SMS_MOCK=true` — SMS не списываются с баланса.
