# Этап I-E — Cloudflare (защита DDoS + DNS)

> Пошагово для новичка. Домен: **Reg.ru** | Сервер: Timeweb VPS `5.129.243.246` | Сайт: https://darom-app.online/

## Что получится

| Было | Станет |
|------|--------|
| DNS только Reg.ru | DNS через **Cloudflare** (бесплатно) |
| Атаки бьют сразу в Timeweb | Cloudflare **фильтрует** лишний трафик |
| DDoS только Timeweb | **Два слоя:** Timeweb + Cloudflare |

**Схема:**

```
Пользователь → Cloudflare (фильтр) → Timeweb VPS → nginx → сайт «Даром»
```

---

## ⚠️ Россия: сайт не открывается без VPN

С **июня 2025** многие российские провайдеры (МТС, Мегафон, Ростелеком и др.) **режут трафик к Cloudflare** (лимит ~16 КБ). С **оранжевым облаком ☁️ (Proxied)** сайт в России часто **не грузится**; **с VPN** — открывается.

**Решение для «Даром» (аудитория в РФ):**

1. Cloudflare → **DNS** → записи `A` для `@` и `www`
2. Нажмите **оранжевое облако** → станет **серым** (**DNS only**)
3. IP остаётся `5.129.243.246` — трафик идёт **напрямую на Timeweb**, минуя прокси Cloudflare

Защита остаётся: **Timeweb DDoS** + nginx (I-D) + rate limit (I-F).  
Прокси Cloudflare для `.ru`-аудитории **не используем**.

Проверка (ПК): `nslookup darom-app.online` → должен показать **`5.129.243.246`**, не `104.21.x.x`.

## Шаг 1 — Аккаунт Cloudflare

1. Откройте https://dash.cloudflare.com/sign-up
2. Зарегистрируйтесь (email + пароль) или **Log in**, если уже есть аккаунт.
3. После входа нажмите **«Add a site»** / **«Добавить сайт»**.
4. Введите: `darom-app.online`
5. Выберите план **Free** (бесплатный) → **Continue**.

**Успех:** Cloudflare предлагает проверить DNS-записи (шаг 2).

---

## Шаг 2 — DNS-записи в Cloudflare

Cloudflare покажет таблицу записей. Должны быть (или добавьте вручную):

| Type | Name | Content | Proxy |
|------|------|---------|-------|
| **A** | `@` | `5.129.243.246` | **DNS only** (серое облако ☁️) — для РФ |
| **A** | `www` | `5.129.243.246` | **DNS only** (серое облако ☁️) |

**Proxied (оранжевое)** — для аудитории **вне РФ**; в России часто **не открывается без VPN**.  
**DNS only (серое)** — для «Даром»: трафик на Timeweb, сайт работает у всех провайдеров РФ.

Лишние записи Reg.ru parking — удалите.

Нажмите **Continue**.

---

## Шаг 3 — NS-серверы (переключение с Reg.ru)

Cloudflare покажет **два NS**, например:

```
ada.ns.cloudflare.com
bob.ns.cloudflare.com
```

(У вас будут **свои** — скопируйте именно те, что показаны.)

### Reg.ru

1. https://www.reg.ru/ → **Домены** → **darom-app.online**
2. **DNS-серверы** / **Делегирование**
3. Выберите **«Свои DNS-серверы»** (не «DNS Reg.ru»)
4. Вставьте **оба** NS от Cloudflare
5. Сохраните

**Успех:** Reg.ru принял NS без ошибки.

⏳ Подождите **15 мин – 24 ч** (обычно 15–60 мин). Cloudflare пришлёт email «Active», когда заработает.

---

## Шаг 4 — SSL в Cloudflare

Когда сайт **Active** в Cloudflare:

1. **SSL/TLS** → **Overview**
2. Режим: **Full (strict)** / **Полный (строгий)**

> На Timeweb уже есть Let's Encrypt — этот режим подходит.

**Успех:** https://darom-app.online/ открывается с замком.

| Действие | Где |
|----------|-----|
| **Защита от DDoS** → Включить | Timeweb → Сервер → Сеть (если ещё не включили) |
| Файрвол: только **80** и **443** | Timeweb → Сеть / Firewall |

---

## Шаг 7 — «Under Attack Mode» (только при атаке)

Cloudflare → **Security** → **Settings** → **I'm Under Attack**

Включайте **только** если сайт реально «лежит» от атаки. В обычном режиме — **выключено** (иначе лишняя проверка для пользователей).

## Если что-то сломалось

| Проблема | Решение |
|----------|---------|
| Сайт не открывается после смены NS | Подождать DNS; проверить A-записи в Cloudflare |
| Ошибка SSL / 525 | SSL → **Full (strict)**; на сервере `curl https://darom-app.online/api/health` |
| Бесконечная проверка Cloudflare | Выключить Under Attack Mode |
| Откат на Reg.ru | Reg.ru → вернуть NS `ns1.reg.ru`, `ns2.reg.ru` |
